## Resumen

Esta Nota Técnica define el nuevo **Evento de Nominación de Factura Electrónica**, que permite a un facturador electrónico asignar un receptor identificado a una factura electrónica que originalmente fue emitida como innominada (D208=5). Se establece la estructura completa del formato XML del evento (`rGEveNom`) con todos sus campos, incluyendo datos de identificación del receptor (RUC o documento de identidad), datos de contacto y dirección.

Se incluye también el campo de tipo de operación (GENFE027), incorporado posteriormente el 04/05/2023, que permite clasificar el tipo de relación comercial en B2B, B2C o B2F, con las validaciones correspondientes para verificar la coherencia entre la naturaleza del receptor y el tipo de operación informado.
